# Optical Flow — Fasciculation Rotation Estimation

> Dense optical flow analysis for estimating per-ring rotational motion from both synthetic fasciculation video and real ALS ultrasound data. Outputs normalized rotation values per concentric ring region, which serve as the ground-truth initialization for the downstream Bayesian detection pipeline.

## Project Description

This repository provides two optical flow scripts that estimate fasciculation-like rotational motion in video data:

- **`optical_flow_als_ultrasound.py`** — Processes real ALS muscle ultrasound video within a defined Region of Interest (ROI). Analyzes 20 concentric ring sizes (radii 5–100 px, step 5) across frames 6–24.

- **`optical_flow_synthetic.py`** — Processes the synthetic fasciculation simulation video generated by the `Fasciculation-Simulation` repository. Analyzes 10 rectangular sizes (10–190 px, step 20) across all 40 frames.

Both scripts use **Farnebäck dense optical flow** (OpenCV) to compute the local curl (rotation component) of the flow field, average it over concentric rectangular regions, apply moving average smoothing, normalize by perimeter, and scale by radius.

The resulting CSV output — normalized rotation values per ring per frame — is used as the **initial rotation angle input** (`Original_Groundtruth_Values.csv`) for the `Fasciculation-Detection-Using-Synthetic-Data` Bayesian BBVI pipeline.

---

## Objectives

1. **Estimate per-ring rotation angles** from dense optical flow using finite-difference curl approximation.
2. **Validate the approach on synthetic data** (known ground-truth rotations) before applying to real ultrasound.
3. **Generate initialization CSV** for downstream Bayesian variational inference.
4. **Visualize rotation trajectories** per ring size across frames.
5. **Export structured CSV results** for further analysis or model input.

## Algorithm Overview

### 1. Dense Optical Flow (Farnebäck)
For each consecutive frame pair `(f_t, f_{t+1})`, dense optical flow `(vx, vy)` is computed using OpenCV's `calcOpticalFlowFarneback`. This returns per-pixel velocity vectors.

### 2. Local Rotation (Curl Approximation)
At each pixel `(r, c)`, the local rotation is approximated as the 2D curl of the flow field using finite differences:

```
rotation(r, c) = [vy(r, c+1) - vy(r, c)] - [vx(r+1, c) - vx(r, c)]
               ≈ ∂vy/∂x - ∂vx/∂y
```

### 3. Per-Ring Averaging
For each ring size `s`, rotation values are summed over a square window of side `s` centered on the ROI center, then divided by `s²` to get the mean rotation.

### 4. Moving Average Smoothing
A window-3 moving average is applied to reduce noise in the per-frame rotation signal.

### 5. Normalization & Radius Scaling
Each rotation value is normalized by the ring perimeter (`8 × s`) and then scaled by radius `s`:

```
final_value = (smoothed_rotation / (8 × s)) × s
            = smoothed_rotation / 8
```

This produces a perimeter-normalized, radius-weighted rotation estimate per ring per frame.

### 6. CSV Export
Results are saved as a structured CSV with columns `Frame`, `Size Xpx` for each ring size.

## Key Differences Between the Two Scripts

| Aspect | ALS Ultrasound | Synthetic |
|--------|---------------|-----------|
| Input video | Real ALS ultrasound | Synthetic simulation |
| ROI cropping | Yes (250–500 x, 200–450 y) | No (full frame) |
| Frame range | 6–24 | 0–40 |
| Ring sizes | 5–100 px, step 5 (20 rings) | 10–190 px, step 20 (10 rings) |
| Flow parameters | `pyr_scale=0.5, levels=3, winsize=15, iterations=20` | `pyr_scale=0.5, levels=3, winsize=15, iterations=5` |
| Output CSV naming | With spaces (frame range in name) | With underscores |

## Known Limitations

- Video paths are hardcoded — update before running locally.
- The curl approximation uses simple first-order finite differences; no sub-pixel accuracy.
- Square ROI windows do not perfectly match annular ring geometry.
- No CLI argument parser — all configuration is done by editing the script.
- Processing is CPU-only (no GPU acceleration needed for optical flow).

## Related Repositories

This repository is **Stage 2** of a three-stage fasciculation analysis pipeline:

**Stage 1 — [Fasciculation-Simulation](https://github.com/ShzdiTrnum/Fasciculation-Simulation)**
> Generates synthetic video of concentric ring rotation with acceleration/deceleration dynamics. Provides the synthetic video input for `optical_flow_synthetic.py`.

**Stage 2 — This repository**
> Applies optical flow to both synthetic and real ultrasound video. Outputs rotation CSV used to initialize Stage 3.

**Stage 3 — [Fasciculation-Detection-Using-Synthetic-Data](https://github.com/ShzdiTrnum/Fasciculation-Detection-Using-Synthetic-Data)**
> Takes the optical flow CSV + video and runs Bayesian BBVI + EM to refine rotation estimates with full uncertainty quantification.

      outputs/
